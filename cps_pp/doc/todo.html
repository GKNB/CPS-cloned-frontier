Some Notes:

The _TARTAN v. PARALLEL issue - expand on this:
When I was replacing _TARTAN with PARALLEL, it was because the piece of code involved concerned only the SCU comms layer, not any other QCDSP-specific aspects of the code. The PARALLEL parts are those which can be run on machines other than QCDSP by using the MPI implementation of the SCU. The other QCDSP specific bits were usually concerned with referencing absolute memory addresses, or filling the CRAM, the circular buffer, and so on. This second class of code always appears to come with a none-QCDSP-specific implementaion for the serial version, and in the portable MPI version this 'serial' code is used instead.

Andrew Jackson - 2001.
