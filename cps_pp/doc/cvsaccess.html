Accessing the UKQCD QCDOC Code Repository

To access the CVS repository, you need an account on trumpton.ph.ed.ac.uk (contact Chris Maynard for this).   (Your account needs to be in the qcdoc group.)To check out the current version of the Columbia code, you need to (in bash):
export CVS_SERVER=/usr/bin/cvs 
export CVS_RSH=ssh
export CVSROOT=:ext:YOURUSERNAME@trumpton.ph.ed.ac.uk:/Disk/ukqcd/QCDOC/CVS 
cvs co cps++
This will then prompt you for your trumpton password, and then check the entire codebase out into a directory called `cps++'.

Submission of Changes

Anyone who has checked out the code in the manner described above has the ability to commit their own changes to the repository.  For the time being, we request that all changes are cleared/submitted by ourselves (via Craig McNeile ) so that we can retain overall control of the repository. Eventually there will (probably) be a committee of people to "approve" submissions, comprising of people from UKQCD, Columbia and BNL.

Using ssh

Using the mechanism descibed above, the secure login process means that you have to type in your password every time you perform a cvs operation, and sometime twice when checking in changes.  This can be avoided by setting up your own ssh-agent to remember your passphrase locally and perform the identity check automatically.
